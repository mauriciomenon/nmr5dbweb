Como operar esta tela

Passos simples para rastrear um mesmo registro em varios bancos.

1) Em Diretorio dos bancos, informe a pasta base que contem as versoes do banco (.accdb/.mdb/.duckdb) ou use o botao Navegar.
2) Se quiser restringir a uma tabela, preencha o campo Tabela (opcional) (ex.: RANGER_SOSTAT).
3) Em Filtros, descreva o registro que procura no formato COL=VAL (ex.: RTUNO=1,PNTNO=2304).
4) Clique em Rastrear registro para ver em quais bancos o registro aparece.
A tabela de resultado mostra, para cada arquivo, a data, o tamanho, a engine, se encontrou o registro, em qual tabela foi encontrado, um resumo dos campos principais e, se algo falhar, a mensagem de erro.

1. Diretorio dos bancos

Defina a pasta base onde estao as versoes de banco que serao analisadas.

Escolha a pasta base onde estao os bancos (.accdb/.mdb/.duckdb). Voce pode digitar o caminho ou usar o botao "Navegar".

2. Escopo e filtros da busca

Especifique a tabela (opcional) e o conjunto de filtros obrigatorios.

Exemplo de tabela: RANGER_SOSTAT. Se vazio, o backend tenta encontrar o registro em qualquer tabela de cada banco.
Exemplo de filtros: RTUNO=1,PNTNO=2304. Formato: COL1=VAL1,COL2=VAL2,... Valores numericos sao detectados automaticamente; use aspas para strings com virgula.
Boas praticas

Como reduzir ruido no rastreio

  • Use a tabela quando o dominio ja estiver conhecido; isso acelera a leitura do resultado.
  • Mantenha filtros pequenos e exatos; cada par COL=VAL ajuda o backend a evitar falsos positivos.
  • Leia a coluna de erro antes de assumir ausencia do registro; as vezes o banco falhou, nao a busca.
  • Combine com a comparacao quando precisar entender em qual versao a mudanca surgiu.
